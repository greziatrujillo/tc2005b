Respuestas a las preguntas

  1. ¿Cuál es la diferencia entre Internet y la World Wide Web?
    2. Internet es literalmente la red de computadoras conectadas. El internet incluye World Wide Web, que son documentos e hipervínculos.

  2. ¿Cuáles son las partes de un URL?
    3. Protocolo, dominio, puerto, ruta, query string, fragmento.

  3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
    4. Estos metodos dejan al servidor saber que es lo que quiere hacer el cliente con el recurso.
    • GET: obtener recurso
    • HEAD: verificar existencia de recurso
    • POST: crear recurso nuevo
    • PUT: reemplazar recurso
    • PATCH: actualizar parte de recurso
    • DELETE: eliminar recurso

  4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?
    5. Se debe usar el método POST ya que el formulario sería un recurso nuevo que es creado. Al momento de ingresar los datos, una página nueva es cargada.

  5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?
    6. Se usa el método PUT ya que se actualiza el URL por completo porque se entra a una página completamente diferente.

  6. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?
    7. Un 200 significa que todo fue exitoso, sin algún error. Los códigos empezando con 2 son de éxito.

  7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?
    8. No necesariamente. Los códigos empezando con 4 significan que hay un error de parte del cliente, por lo que el servidor está bien, pero el cliente debe hacer algo para poder acceder. Por lo tanto, como el código es 404 puede ser que el recurso no existe. El cliente tal vez ingresó datos incorrectos o el servidor no está funcionando correctamente.

  8. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?
    9. En el caso de los códigos empezando con un 5, es responsabilidad del desarrollador corregir algún error. En el caso de un 500, significa que hay algo mal con el servidor, por lo que el desarrollador debe investigar y corregir.

  9. ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.
    10. Un atributo depreciado significa que son reemplazados por otros atributos ya que no se usan actualmente. Por ejemplo, en HTML4 se usan los atributos longdesc y abbr mientras que en HTML5 son depreciados.

  10. ¿Qué componentes de estructura y estilo tiene una tabla?
    11. En una base de datos, las tablas se usan para organizar datos en forma de columnas y renglones.

  11. ¿Cuáles son los principales controles de una forma HTML5?
    12. Algunos de los controles principales incluyen:
    • campos de texto <*input> que permite el usuario introducir datos
    • readonly no permite usuario modificar valores despues de mandar formulario
    • placeholder es texto dentro de caja de permiso para explicar el proposito
    • type es el tipo de valor que puede ser ingresado

  12. Sobre el ciclo de vida y desarrollo de los sistemas de información: ¿Cuál es el ciclo de vida de los sistemas de información? ¿Cuál es el ciclo de desarrollo de sistemas de información?
    13. El ciclo de vida del desarrollo de los sistemas de informacion se divide en 7 fases:
    1. planificacion - Identificar requisitos que se piden
    2. analisis - requisitos identificados son analizados
    3. diseño- el sistema es planeado y estructurado
    4. desarrollo - En base al diseño, el sistema es creado.
    5. prueba- Inspeccion exhaustiva del sistema.
    6. Integracion - lanzamiento de sistema.
    7. Mantenimiento - Monitorear y actualizar sistema.

  1. Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?
    2. El uso de !important debe ser usado con mucho cuidado, ya que usarlo de manera excesiva puede causar problemas dentro de la página en la que se usa CSS. Ya que !important le da prioridad a un atributo o sus reglas, hay que crear un jerarquia de estilos para identificar lo más importante que se debe plasmar.

  2. Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?
    3. Aparte de que puede tener derechos de autor, la imagen puede afectar el rendimiento del sitio o igualmente queda mal puesta la imagen, por lo que no queda bien como fondo de página. Por ejemplo, si se escoge ina imagen de 120px x 100px y la página es de 500px x 500px, obviamente no va quedar bien.

  3. Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?
    4. Las propiedades pt y px son fijas, por lo que su temaño identificado no se ajusta al momento de cambiar el tamño de la pantalla. Del otro lado, % se considera una propiedad responsiva, por lo que se ajusta al tamaño de la pantalla. Es mejor usar % cuando se prioriza un sitio responsivo o que se va cambiar a una aplicación móvil.

  4. ¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?
    5. Para poder mejorar el rendimiento de un sitio, normalmente es necesario simplificar el código usado. Minimizando el CSS ayuda en la eliminación de detalles inecesarios como los espacio, saltos, comentarios, carácteres y más detalles aún. Con la eliminación de estos, el código es literalmente minimizado y ayuda el rendimiento.
